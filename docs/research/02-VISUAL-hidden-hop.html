The Hidden Hop クエリに見えない2ホップ目を、LLMなしでどう拾うか

マルチホップ検索の失敗は「選択」でなく「候補集め」で起きる。その正体は クエリに語彙的にも意味的にも似ていない2ホップ目の正解＝hidden hop。 このページは、LinearRAG から出発したこの研究が今どこにいるかを、図で辿れるようにしたものです。

制約: トークンフリー（LLM不使用） 制約: 訓練フリー 評価: 2Wiki · HotpotQA · MuSiQue（各1000問）

Section 1 — これまでの道のり

LinearRAG から始まり、5回脱皮した

出発点は既存手法 LinearRAG（固有名詞グラフ＋PPRの軽い多ホップ検索）を 「改良して論文を書く」こと。毎回死んだのは手法の主張で、生き残って強くなったのは診断。 番号は実際に起きた順序です。

0

LinearRAG を直接改良（coverSoft）

DIEDクロスエンコーダに負け／SETR が先行研究／GPT-4o が上限を示し、手法レースが三方向で閉じた。

1

問いを転換 → Hidden Hop を発見

SURVIVED失敗の正体は「クエリに似てない2ホップ目の正解」。それを測る物差し hiddenness指標を作った。この診断レンズが以後ずっと背骨になる。

2

グラフを「検索器 → 注入器」に読み替え

SURVIVEDグラフPPRだけが hidden hop を候補プールに注入できる（dense / RM3 / entity-ANN に勝利）。

3

SPRIG対決で前提が崩壊 → もっと強い絵へ

DIED「グラフは検索器として弱い」という前提。SURVIVED 棲み分けの法則（言葉検索＝可視の王／エンティティPPR＝隠れの王）と、 種の置き方で隠れ文書への到達が決まる機序（固有名詞種は届く／キーワード文書種は構造的に届かない）。

4

選択もトークンフリーに / システム化

SURVIVED可視/隠れを混ぜず併合する min-rank（CombMAX）で選択。 融合(min-rank)が、正しく調整した LinearRAG(λ0.05)＝トークンフリー枠の最良を全データで上回る （GoldR@5 +6.1/+28.6/+11.7）。候補生成が現行LLMを +34.7pt 押し上げ。 ※「HippoRAGに並ぶ」は見出しにしない（LinearRAG自身が主張済み＋現行のHippoRAG 2には大差負け）。

5

λ問題の発覚と「自滅」主張の撤回（現在地）

RETRACTED「LinearRAG が DPR 混合で自滅」は我々の実装バグ（λ を論文の 0.05 でなく 1.5 に誤設定）だった。原論文を精読し撤回。SURVIVED診断・融合・ 候補生成の価値・d^k 減衰は λ 非依存で無傷。物語は「発見」から「計器＋機序＋融合システム」に収束。

要するに

「LinearRAGを良くする研究」から出発し、いまは トークンフリー多ホップ検索の失敗を診断し、可視/隠れの棲み分けで組み直す研究に化けた。 LinearRAG 自身は「出発点の乗り物」であり、正しく調整すれば健全な単一レジーム手法（可視偏重データでは弱い）。

Section 3 — 中心概念

Hidden Hop とは何か

質問「Lothair II の母はいつ死んだ？」は2つの根拠を要する。1ホップ目は取りやすい。 2ホップ目は橋エンティティ経由でしか繋がらず、クエリと似ていない。

Section 4 — 物差し

hiddenness指標：隠れ度を測る

「どのクエリ基準手法から見ても似ていない」を [0,1] で数値化する。0＝丸見え、1＝完全に隠れている。

hidden(g) = 1 − ½ · ( Pdense(g) + Pbm25(g) )
  Pdense(g) = 意味類似(埋め込み内積)でのパーセンタイル順位
  Pbm25(g) = 語彙一致(BM25)でのパーセンタイル順位
  質問単位: hidden(q) = maxg∈gold hidden(g)  （最も隠れた1件が律速）
hidden ≈ 0  可視（クエリに似てる）隠れ（橋の先）  hidden ≈ 1

設計のキモ ①dense と BM25 の両方を使う＝片方だけでは取り逃す隠れ方を捉える。 ②絶対スコアでなくパーセンタイルにして、スケールの違う2信号を公平に平均。 ③質問集約は max＝AllGoldHit（全正解が揃う）は最も隠れた1件で決まるから。

Section 5 — 仕組み

グラフと純エンティティPPR：どこに水を注ぐかが全て

グラフは二部構造。段落と固有名詞を、「その段落がその固有名詞に言及していたら」線で繋ぐ。 段落同士は直接繋がらない——必ず固有名詞を経由する。PPRは「種に水を注ぎ、線を伝って染み出させ、常に種へ引き戻す」計算です。

x(t+1) = (1−d)·r + d·W·x(t)
  r = 種の分布（正規化） · W = 二部グラフの遷移行列 · d = 0.5
  展開すると  x ≈ (1−d)r + d(1−d)Wr + d²(1−d)W²r + …
  → Wᵏr の項が「k ホップ先」に対応（dᵏで減衰しつつ橋を渡る）

種を段落に置くと隠れ段落へ届かない——SPRIGのHybrid（BM25上位を種）は段落に注いで動けない （上位50件＝種と100%一致、実測）。固有名詞に注げば橋づたいに隠れ段落へ届く。 ※かつて「LinearRAGはクエリ類似を混ぜて流れを薄めていた」と書いたが、これは我々がλを過大設定した バグ由来の誤りで撤回済み（脱皮5参照）。本家はλを小さく設計しており自滅しない。

Section 8 — 最前線

本当のボトルネックは選択でなく候補生成

50件から最終5件を選ぶ選択も試した（枠で分けるスロット／パラメータフリーの min-rank＝ 「順位を足さず良い方で入場」）。だが現行の安いLLM(gpt-5.4-mini)を参照に置くと、もっと大事なことが見えた—— 同じLLMでも、候補プールが貧しいと 43、我々のトークンフリー融合プールを与えると 77.7 に跳ねる。 指標は AllGoldHit@5（2Wiki, 300問, 全て同一subset）。

3つの差分がそのまま主張になる：①候補生成の価値はデータの隠れ度に比例——同じLLMが我々の融合プールで 2Wiki +34.7 / MuSiQue +12.7 / Hotpot +6.7 跳ねる（隠れが多いほど効く、診断が予言する並び）。 ②完全無料 > 現行LLM+素朴検索（隠れ多の2Wiki・MuSiQueで無料min-rankがLLM@BM25に勝つ； 可視多のHotpotは負け、正直に）。③選択自体はLLMが上（同一プールで一貫、+21〜23）。 枠分け(min-rank)は「順位を足す混合」より良い（隠れホップを埋めない）が、読解勝負ではLLMに及ばない。 勝負どころは選択でなく候補生成だった。

正直な現在地（型番に強い主張へ）

「トークンフリー選択がLLMに勝つ」は現行モデルでは成立しない（脆い主張だった）。代わりに残るのは 「マルチホップの真のボトルネックは候補生成。それはトークンフリーに解け、我々の融合プールは最強クラスの LLM選択器を +35 押し上げる」——新しいLLMほど良いプールを欲しがるので、時代が進むほど価値が上がる。

Section 9 — 立ち位置

先行研究との距離

LinearRAGarXiv 2510.10114
出発点の乗り物。固有名詞グラフ＋PPR。論文値(λ0.05)なら健全な単一レジーム手法だが、可視偏重データ(hotpot)で弱く、直交レジームの融合(min-rank)が全データで上回る（GoldR@5 +6〜+29）。
HippoRAGNeurIPS 2024
エンティティPPRで多ホップ＝「純エンティティPPR」の思想の源流。KG構築にLLM。旧版(2024)とは互角圏だが、現行のHippoRAG 2(2025, R@5 90.4/96.3/74.7)には全データ大差負け＝LLM構築グラフの価値は残る（正直に）。
SPRIGarXiv 2602.23372
NER共起グラフ＋PPR、CPU-only。グラフを"検索器"として使い、種の置き方で損をしていた。我々は棲み分けを診断し"注入器＋融合"に。
StratRAG2026
「橋エンティティはクエリに現れない」と現象を観察。我々はそれを測り(hiddenness)、機序を与え、解いた。

新規性の重心

エンティティPPRや融合そのものは既出（HippoRAG/SPRIG）。我々の新しさは「hiddenness指標という計器」＋ 「失敗の解剖（候補生成の崖・可視/隠れの鏡像）」＋「直交レジームの融合が正しく調整した単一手法(LinearRAG λ0.05)を超える実証」——診断が主役、融合システムが従。
