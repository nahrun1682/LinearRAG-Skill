# Stage 2 が律速 — 活性化カバレッジ貪欲選択（次の一手）

2026-07-03 ・ 探針 `research/rank_headroom.py`（baseline, canonical corpus, sm NER, 300q）

## 動機

`ResiRAG`（Stage 1 = 活性化伝播の改良）は正典コーパスでも純増が小さく、gold-recall指標で
AllGoldHit がほぼ動かなかった（[eval-basis](2026-07-03-eval-basis.md)）。
「ブリッジを活性化できても2つ目の根拠が top-5 に入らない」= ボトルネックが Stage 2（最終ランキング）へ
移った、という仮説を数字で検証する。

## 実測: Stage 2 に大きな伸び代がある

top-50 まで掘って gold パッセージの順位分布を測定（baseline, delta=0.8）。

| 指標 | MuSiQue | 2wiki |
|---|---|---|
| GoldRecall@5 | 34.3% | 51.2% |
| AllGoldHit@5（現在） | **7.0%** | **28.7%** |
| AllGoldHit@10 | 14.7% | 35.3% |
| AllGoldHit@20 | 22.3% | 43.3% |
| AllGoldHit@50 | 40.0% | 51.7% |
| 見つかった gold の最悪順位: 中央値 / p75 / p90 | 19 / 32 / 45 | 4 / 14 / 29 |
| **完璧な top-20→top-5 再ランカーの上限** | **20.3%（約3倍）** | **40.3%（+11.6pt）** |

読み取り:

1. **2つ目の gold は「取れていない」のではなく rank 6〜20 に埋まっている**。
   MuSiQue の worst-gold-rank 中央値は 19 位。Stage 1 をいくら改良しても top-5 カットで落ちる。
   ResiRAG が頭打ちだった理由の説明にもなる。
2. **partial@5**（gold の一部だけ top-5 に入る質問）が MuSiQue 60.7% / 2wiki 49.7%。
   このうち全 gold が top-20 内にいるものが両データセットで約23% ＝ 再ランキングで直せる母集団。
3. **決定的: 冗長性の実測**。top-5 のパッセージのうち **MuSiQue 75% / 2wiki 84% が、
   より上位のパッセージとエンティティを共有**している。top-5 が「hop 1 の同じ話の言い換え」で
   埋まり、hop 2 の証拠を押し出している。

原因は構造的: **AllGoldHit は集合の指標なのに、最終 top-5 カットが 1 本ずつの点数順**で、
パッセージ間の相補性を一切見ていない。

## 提案手法: 活性化カバレッジ貪欲選択（Coverage-Greedy, Stage 2）

top-N（N=20〜50）候補から、点数順でなく**限界利得の貪欲法**で top-k 枠を埋める:

```
gain(p | 選択済みS) = α·関連度(p)
                     + Σ_{e ∈ entities(p) かつ e が S に未被覆} activation[e] / level[e]
```

- 「まだ被覆していない活性化エンティティを新しくどれだけ覆うか」で次の1枠を選ぶ。
  同じ話の反復は限界利得ゼロになり、ブリッジ側の証拠が自然に浮上する。
- カバレッジ項は単調劣モジュラなので貪欲法に (1−1/e) 近似保証。
- 計算は候補パッセージの疎行列行参照だけ。**トークンフリー・ほぼゼロコスト**。
- Stage 1 と地続き: activation / level (L_ei) をそのまま流用（式7と同じ量）。

## 一貫した原理としての位置づけ（論文の背骨）

Stage 1 残差伝播（[research-direction](2026-07-03-research-direction.md)）と合わせると、
「**検索の状態で次の判断を条件付ける**」という単一原理が両ステージを貫く:

- Stage 1: これまでカバーしたクエリ成分（残差）で次ホップの σ を条件付ける
- Stage 2: これまで選んだパッセージ集合で次の 1 枠の利得を条件付ける

両方ともトークンフリー・訓練フリー・行列演算のみ。

## 既存研究との差分

| 手法 | 多様化の基準 | LLM/訓練 | うちとの差分 |
|---|---|---|---|
| **MMR** (Carbonell&Goldstein 1998) | 埋め込みの汎用類似度 | 不要 | 埋め込み多様性でなく**グラフ活性化質量のカバレッジ**。L_ei で Stage 1 と接続。**対照実験で MMR を同条件で回し、活性化カバレッジが上回るか**を示す＝「グラフが効く」証明 |
| **BDTR** (2509.25530) | ブリッジ証拠を上位へ | LLM思考生成 | 問題意識（順位押し上げ）は同じ、処方が構造のみ |
| **BridgeRAG** (2604.03384) | ブリッジ条件付きペア選択 | LLM judge必須 | オンライン LLM ゼロ |
| **PropRAG** (EMNLP25) | 命題パスの beam search | LLM命題抽出 | 表現側も LLM フリー |
| Set-based IR / facility location | 被覆最大化 | — | 被覆の重みを**クエリ活性化**に置く点が新規 |

## 成功基準（先験的に宣言）

- MuSiQue AllGoldHit@5: 7.0% → **14% 以上**（再ランキング上限 20.3% の半分超）
- 2wiki AllGoldHit@5: 28.7% → **34% 以上**
- 同条件の MMR 対照を上回る
- レイテンシ増ほぼゼロ（候補は既に計算済みの PPR スコア上位）

## 実験順序

1. sm/trf 較正（2wiki trf ビルド）を先に確定
2. `research/` に coverage-greedy 実装（skill 本体は不変）
3. MMR 対照 + α・N のグリッド探索（MuSiQue/2wiki 300q）
4. 3データセット + JEMHopQA 一斉評価（gold-recall）
5. ResiRAG(Stage 1) との合成 — 単一原理が両ステージで効くか
