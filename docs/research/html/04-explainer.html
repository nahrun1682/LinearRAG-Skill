マルチホップ検索は、どこで失敗し、無料でどこまで解けるか

複数の文書をたどらないと答えられない質問で、RAGの検索はしばしば失敗する。 本研究は失敗の原因を測り、機序を説明し、 LLM呼び出しゼロの検索でどこまで解けるかを見極め、 「安く解ける部分」と「LLMにしか買えない部分」の境界を実測した。

失敗の正体は「選び方」でなく「候補集め」——質問に映らない“隠れ文書”がそもそも候補に入らないこと。 それはトークンフリーに解けるが、その先（読解・深さ・関係）はLLMの価値が残る。

§1 問題

2ホップ目は、質問に映らない

RAGはLLMに答えさせる前に関連文書を検索して渡す。検索の質が回答の上限を決める。 検索は2段階——数万から50件に絞る「候補集め」と、そこから5件を選ぶ「選択」。 マルチホップ質問「曲 Changed It の歌手の出身地は？」は、①曲のページで歌手（Nicki Minaj）を知り、 ②歌手のページで出身地を知る——両方揃って初めて答えられる。問題は2ホップ目だ。

§2 計器

「隠れ度」を連続値で測る

まず、正解文書がどれだけ質問から見えないかを数値化する物差しを作った。 2種類の検索（意味・語彙）の両方から見て、コーパス全体でどれだけ下位に沈むか:

hidden(g) = 1 − ½·( P意味(g) + P語彙(g) ) 　P=順位パーセンタイル、質問単位は最も隠れた正解のmax、四分位で層別
hidden ≈ 0 可視隠れ（橋の先） hidden ≈ 1

先行研究には「2ホップ目が質問に出るか」の二値分類はあったが、 連続の計器にしたことで「どのくらい隠れているとどの手法が死ぬか」を層別できる。これが以後の全実験のレンズになる。

§3 診断

失敗は「選択」でなく「候補集め」

この計器で質問を隠れ度の四分位に分け、様々な手法を層別すると、全手法が同じ崖から落ちる（2Wiki）:

隠れ度キーワード検索小型AI再選択GPT-4oに選ばせる
可視85%100%91%
やや隠れ47%64%75%
隠れ9%11%15%
深く隠れ0%0%0%

深く隠れた文書では、GPT-4oに選ばせても0%。原因は単純で、 正解が候補50件にそもそも入っていない。賢い選択器も、候補に無いものは選べない。

診断

マルチホップ検索のボトルネックは、50→5件の選択ではなく、数万→50件の候補集めにある。以後、照準をここに合わせる。

§4 機序

グラフの多段伝播だけが、橋を渡って隠れ文書に届く

隠れ文書に届く唯一の安価な道具＝グラフ検索。文書と固有名詞を「言及」で結んだ地図の上で、 質問の固有名詞から水を染み出させる（PPR）。水は文書→固有名詞→文書と橋を渡り、 クエリと似ていない2ホップ先にも届く——キーワード検索や意味検索の「クエリとの類似度で並べる」やり方では、 原理的に届かない領域だ。

この注入力を安価な手法で比べると差は桁違い（深く隠れた質問での全正解被覆率）:

2Wiki

MuSiQue
29% / 6% / 0%

深隠れビンでは、グラフ注入だけが隠れホップをプールに届ける（意味検索・テキスト拡張は届かない）。

§5 システム

直交レジームを混ぜず併合する

隠れ度で層別すると、キーワード検索とエンティティ伝播は鏡像——前者は可視の王で隠れで全滅、後者は逆。 どちらも単一では必ずどこかで崩れる。だから混ぜず併合する: min-rank（各候補を「良い方の順位」で採用。順位を足すと隠れホップが沈む＝古典CombMAX）。

§7 貪欲な限界探索

SOTAを本気で狙い、境界を実測した

現行LLM-SOTA（HippoRAG 2）との残差を、トークンフリーで貪欲に埋めにいった。 各方向に成功基準とキル基準を事前宣言し、感情でなく数字で進退を決めた（λ問題で学んだ規律）。3方向すべてが壁に当たった:

選択（Hotpot）

6敗

プールは飽和(94.6)、LLMなら82.7出るのに安価選択は61.7。ペア選択も即死。トークンフリー選択はset-aware読解に届かない。

深さ（MuSiQue）

プール止まり

反復再シードで4hopプール10→15と機序は効くが、R@5に変換されない（深い正解がプールに入っても top-5 に届かない）。

関係（2Wiki）

天井僅少

到達オラクル: 外した隠れ正解の55%は橋がグラフに無い、45%もハブ経由(df中央値680)。関係ラベルは刺さらない。

境界地図（この探索の成果）

候補生成は安く解けた。選択・深さ・関係の残差は、読解/推論を要する＝LLMが実際に買っているもの。 貪欲に全方向を殴ったからこそ、この境界が数字で引けた。失敗も「境界の実測」として論文の芯になる。

§8 これは上限か

「トークンフリーの上限」ではない — 我々の構成の、安価な改良の限界

正確に線を引く。我々が示したのは「LinearRAG流の共起グラフ＋2021年の埋め込み＋単一パスPPRという1構成での、安価な改良の限界」。分野の上限ではない。

上限に近い（本物の壁）

  • トークンフリー＆訓練フリーの“選択”——6敗＋構造的理由（pointwise寄りで set-aware 読解ができない）。LLMの価値が最も置き換えにくい。

上限ではない（残るヘッドルーム）

  • エンティティリンク品質——外した55%は「橋がグラフに無い」＝NER/正規化の失敗。coref・別名リンクは全部TFで可能。次に登るならここ。
  • 現代の埋め込み器（bge-m3等、ローカル＝TF）。可視側もリンク側も底上げされうる。
  • 別のグラフ構造（文グラフ・ハイパーグラフ・命題グラフ）。競合が今まさに登っている。
  • 反復検索を精緻に（我々の再シードは粗い1周だけ）。

だから論文の主張は「トークンフリー打ち止め」ではなく—— 「候補生成は安価に解ける。LLM-SOTAとの残差は読解/選択に集中し、そこが最も置き換えにくい。残る安価なレバー（リンク品質・現代埋め込み・別構造・反復）は future work」。 査読で「bge-m3使えば？」と刺されても、明示的に範囲外と置いてあるので受けられる。

§9 総括

何が新しく、何が限界か

既知（先行研究）

  • 橋質問が最難という観察（StratRAG）
  • 可視/不可視の二値分類（Regime-Conditional 2026）
  • エンティティPPRが多ホップに効く（HippoRAG/SPRIG）
  • 「トークンゼロでHippoRAG級」（LinearRAG自身）／グラフ×語彙融合（SPRIG等）
  • MAX型順位併合（CombMAX 1994）

本研究の新規性

  • 連続の隠れ度計器と失敗の層別（GPT-4oすら深隠れ0%の崖）
  • ボトルネックの局在（選択でなく候補集め）
  • グラフ多段伝播の注入優位（深隠れで51% vs dense 4%／RM3 3%）
  • 候補の価値の定量（同一現行LLMで+34.7、隠れ度に比例）
  • 安く解ける/LLMの価値の境界地図（選択・深さ・関係の限界を機序込みで実測）

限界（正直に）

① 現行LLM-SOTA（HippoRAG 2: R@5 90.4/96.3/74.7）には全データ大差負け＝LLM構築グラフ・読解の価値の定量。 ② 選択はLLMが上（同一候補で+21〜23）。③ 4ホップで全手法劣化（グラフ到達のdk減衰、最難正解の順位 16→268→1893）。 ④ 隠れ度指標は使用する埋め込み・BM25に相対的。⑤ 純エンティティPPRは単独最強ではなく、 融合の相方として最良（RRFとの直交性が高い、経験的観察）。

診断 → 機序 → 処方 → 検証 → 境界の地図。 「大発見」ではないが、嘘が一つもなく、道具と説明と限界の揃った研究になった。
