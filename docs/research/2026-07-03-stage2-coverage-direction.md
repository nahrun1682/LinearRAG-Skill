# Stage 2 が律速 — 活性化カバレッジ貪欲選択（次の一手）

2026-07-03 ・ 探針 `research/rank_headroom.py`（baseline, canonical corpus, sm NER, 300q）

## 動機

`ResiRAG`（Stage 1 = 活性化伝播の改良）は正典コーパスでも純増が小さく、gold-recall指標で
AllGoldHit がほぼ動かなかった（[eval-basis](2026-07-03-eval-basis.md)）。
「ブリッジを活性化できても2つ目の根拠が top-5 に入らない」= ボトルネックが Stage 2（最終ランキング）へ
移った、という仮説を数字で検証する。

## 実測: Stage 2 に大きな伸び代がある

top-50 まで掘って gold パッセージの順位分布を測定（baseline, delta=0.8）。

| 指標 | MuSiQue | 2wiki |
|---|---|---|
| GoldRecall@5 | 34.3% | 51.2% |
| AllGoldHit@5（現在） | **7.0%** | **28.7%** |
| AllGoldHit@10 | 14.7% | 35.3% |
| AllGoldHit@20 | 22.3% | 43.3% |
| AllGoldHit@50 | 40.0% | 51.7% |
| 見つかった gold の最悪順位: 中央値 / p75 / p90 | 19 / 32 / 45 | 4 / 14 / 29 |
| **完璧な top-20→top-5 再ランカーの上限** | **20.3%（約3倍）** | **40.3%（+11.6pt）** |

読み取り:

1. **2つ目の gold は「取れていない」のではなく rank 6〜20 に埋まっている**。
   MuSiQue の worst-gold-rank 中央値は 19 位。Stage 1 をいくら改良しても top-5 カットで落ちる。
   ResiRAG が頭打ちだった理由の説明にもなる。
2. **partial@5**（gold の一部だけ top-5 に入る質問）が MuSiQue 60.7% / 2wiki 49.7%。
   このうち全 gold が top-20 内にいるものが両データセットで約23% ＝ 再ランキングで直せる母集団。
3. **決定的: 冗長性の実測**。top-5 のパッセージのうち **MuSiQue 75% / 2wiki 84% が、
   より上位のパッセージとエンティティを共有**している。top-5 が「hop 1 の同じ話の言い換え」で
   埋まり、hop 2 の証拠を押し出している。

原因は構造的: **AllGoldHit は集合の指標なのに、最終 top-5 カットが 1 本ずつの点数順**で、
パッセージ間の相補性を一切見ていない。

## trf 索引での再測定（2026-07-03、決定版・300q）

上表は sm 索引。評価基盤を trf に統一（[eval-basis](2026-07-03-eval-basis.md)）した後で測り直した。

| 指標 | MuSiQue | 2wiki | HotpotQA |
|---|---|---|---|
| AllGoldHit@5（現在） | 9.7% | 36.3% | 31.3% |
| AllGoldHit@20 | 26.3% | 50.0% | 64.0% |
| AllGoldHit@50 | 40.7% | 59.0% | 76.3% |
| worst-gold-rank 中央値 | 16 | 4 | 8 |
| **top-20→top-5 再ランカーの上限** | **23.3%（+13.6pt, 2.4x）** | **49.0%（+12.7pt）** | **55.3%（+24.0pt）** |
| 冗長性（top-5 が上位とエンティティ共有） | 71% | 83% | 87% |

**trf でも（むしろ trf の方が）Stage 2 の伸び代は大きい**。sm→trf で NER が良くなり
候補プールに 2 つ目の gold が入る率（recall@20）が上がったため、2wiki の上限は 40.3%→49.0%、
MuSiQue も 20.3%→23.3% に拡大。冗長性は 71〜87% で高いまま＝被覆選択の攻撃対象が健在。

**GO 判断**: Stage 2 の活性化カバレッジ貪欲選択を実装する。伸び代（上限−現在）は
HotpotQA +24.0pt / 2wiki +12.7pt / MuSiQue +13.6pt。貪欲法で上限の 30〜60% を取ると仮定すると、
AllGoldHit@5 の予想改善は HotpotQA +7〜14pt / 2wiki +4〜8pt / MuSiQue +4〜8pt。

## 提案手法: 活性化カバレッジ貪欲選択（Coverage-Greedy, Stage 2）

top-N（N=20〜50）候補から、点数順でなく**限界利得の貪欲法**で top-k 枠を埋める:

```
gain(p | 選択済みS) = α·関連度(p)
                     + Σ_{e ∈ entities(p) かつ e が S に未被覆} activation[e] / level[e]
```

- 「まだ被覆していない活性化エンティティを新しくどれだけ覆うか」で次の1枠を選ぶ。
  同じ話の反復は限界利得ゼロになり、ブリッジ側の証拠が自然に浮上する。
- カバレッジ項は単調劣モジュラなので貪欲法に (1−1/e) 近似保証。
- 計算は候補パッセージの疎行列行参照だけ。**トークンフリー・ほぼゼロコスト**。
- Stage 1 と地続き: activation / level (L_ei) をそのまま流用（式7と同じ量）。

## 実験結果（2026-07-03、trf, 300q, top_n=50）— `research/stage2_experiment.py`

3 つの選択戦略を**同じ top-50 候補集合**（元 LinearRAG の PPR で生成）で比較。

| 選択戦略 | 2wiki R/AGH | musique R/AGH | hotpot R/AGH |
|---|---|---|---|
| score（元 LinearRAG＝点数順） | 63.5 / 36.3 | 37.8 / 9.7 | 53.5 / 31.3 |
| **coverDense（提案）** | **66.4 / 35.3** | **43.9 / 10.3** | **67.5 / 40.3** |
| denseMMR（対照: 密+埋め込み多様性） | 53.8 / 20.0 | 36.0 / 5.7 | 64.7 / 37.7 |
| dense（密のみ） | 54.0 / 22.7 | 37.1 / 7.7 | 65.3 / 38.3 |
| cover（当初案: PPR+活性化被覆） | 63.9 / 36.3 | 38.8 / 9.7 | 54.2 / 31.7 |
| mmr（元 PPR+埋め込み多様性） | 56.0 / 24.7 | 31.9 / 2.3 | 46.2 / 21.7 |
| oracle（上限） | 80.8 / 59.0 | 69.7 / 40.7 | 88.0 / 76.3 |

（各セル GoldRecall@5 / AllGoldHit@5）

### 読み取り

1. **当初案（PPR 上の活性化被覆 = `cover`）は不発**。baseline と同点。診断（`/tmp/stage2_diag.py`）で、
   見逃し gold の 47% は「未被覆の活性化エンティティが 0」＝識別特徴が**答え**（クエリ時に未活性化）
   のため、活性化信号では距離が測れない。当初仮説は棄却。
2. **MMR（埋め込み多様性）はマルチホップに逆効果**（全データセットで悪化）。gold 同士は意味が
   近く、多様性が 2 つ目を追い出す。
3. **効いた形は `coverDense`**＝「密類似度でアンカー ＋ 活性化 IDF 被覆で重複除去」。
   見逃し gold は密類似度では 77 パーセンタイル（PPR が不当に沈めている）ことに着目した設計。
   - GoldRecall@5 が baseline 比 **2wiki +2.9 / musique +6.1 / hotpot +14.0**。
   - AllGoldHit@5 は hotpot **+9.0**、musique +0.6、2wiki −1.0（まだら）。
4. **新規性の証明（対照実験）**: `coverDense` は `denseMMR`（同じ密アンカー＋埋め込み多様性）を
   全データセットで大きく上回る（2wiki +12.6 / musique +7.9 / hotpot +2.8 GoldRecall）。
   さらに `denseMMR` は素の `dense` より悪い。つまり**「重複除去に効くのは汎用の多様性でなく、
   グラフの活性化被覆」**が実証された ＝ グラフ信号が価値を運んでいる。
5. hotpot で最大効果（元の PPR が密に負けているデータ＝密アンカーの伸びしろが大きい）、
   2wiki で最小（元 PPR が既に密より強い）。

### 対照実験: 効いているのは「汎用の重複除去」でなく「グラフ活性化信号」（300q, GoldRecall@5）

「ただ dense が強い / どんな除去でも効くのでは」という批判を潰すための対照。同じ密アンカー上で
被覆の重み信号だけを変える。

| 被覆信号 | 2wiki | musique | hotpot |
|---|---|---|---|
| **coverDense**（活性化 × IDF 重み） | **66.4** | **43.9** | **67.5** |
| coverUniform（活性化エンティティ・均等重み） | 63.0 | 40.0 | 53.8 |
| coverAll（全エンティティ・均等＝汎用除去） | 35.3 | 27.1 | 42.8 |
| （参考）dense（除去なし） | 54.0 | 37.1 | 65.3 |
| （参考）score（baseline） | 63.5 | 37.8 | 53.5 |

- **3 データセットで単調に coverDense > coverUniform > coverAll**。
- **coverAll（汎用エンティティ重複除去）が全手法中で最悪**（全 passage が共有するシードまで
  多様化しようとして関連 passage を押し出す）→ 「any dedup works」を完全否定。
- **活性化エンティティへの限定だけで激変**（coverAll→coverUniform、2wiki +27.7pt）＝
  クエリ活性化の構造that's essential。
- **活性化重み(IDF)がさらに上乗せ**（coverUniform→coverDense、hotpot +13.7pt）。
- 「ただ dense が強い」も否定: dense 単体は 2wiki/musique で baseline に負ける（54.0<63.5,
  37.1<37.8）が coverDense は勝つ。coverDense は dense 単体も常に上回る（+11.4/+6.5/+1.6）。

### full 1000q 確定（coverDense@1.0 vs baseline）

| データセット | GoldRecall@5 | AllGoldHit@5 |
|---|---|---|
| 2wiki | 63.1 → **65.2**（+2.1） | 35.2 → 33.7（−1.5） |
| musique | 37.4 → **43.7**（+6.3） | 11.0 → 11.2（+0.2） |
| hotpot | 51.8 → **65.5**（+13.7） | 29.5 → **39.7**（+10.2） |

### AllGoldHit まだらの機序（GoldRecall↑ / AllGoldHit↓ の共存）

AllGoldHit は「両 gold 揃い」の全か無か。coverDense は被覆多様性を優先するため、
(a) rank 6+ の 2 つ目 gold を浮上させ recall を上げる一方、(b) **既に揃っているペアの
片方を、g1 とエンティティ共有＝低被覆利得と判定して降格**させ、揃いを壊すことがある。
baseline が強い 2wiki は壊す対象（完成ペア）が多く微減、baseline が弱い hotpot は
作る余地が大きく +10。→ 直し方: α 増、top-1〜2 の PPR pick 固定、降格ペナルティ。

### 完成ペア問題の修正: ソフト飽和被覆（A/B/C 比較, 300q, top_n=50）

pair-breaking を原理的に直す 3 候補を比較（恣意的なランク固定 top-1〜2 は棄却）。

| 手法 | 2wiki R/AGH | musique R/AGH | hotpot R/AGH |
|---|---|---|---|
| score（baseline） | 63.5 / 36.3 | 37.8 / 9.7 | 53.5 / 31.3 |
| coverDense（前ベスト） | 66.4 / 35.3 | 43.9 / 10.3 | 67.5 / 40.3 |
| **A: coverSoft γ=0.5/0.3** | **67.0 / 36.3** | **44.8 / 10.3** | **68.8 / 43.0** |
| B: coverDual（PPR+密アンカー） | 65.7 / 34.7 | 41.9 / 9.3 | 62.8 / 38.3 |
| C: residual@0.3（残差スロット） | 56.3 / 23.7 | 37.0 / 6.0 | 66.8 / 39.3 |

- **A（ソフト飽和被覆）が採用**。エンティティを m 回目に覆う利得を `w_e·γ^m`（γ≈0.3〜0.5）に。
  橋エンティティが 2 つ目 gold でも部分点を保つので**完成ペアが引き剥がされない**。
  - coverDense を全データセットで上回り、**2wiki の AllGoldHit 退行が消滅**（35.3→36.3＝baseline）、
    **hotpot AllGoldHit は +2.7 さらに上昇**（40.3→43.0）。γ=0.3/0.5 で安定（脆くない）。
- **B（二重アンカー）不採用**: PPR の弱さを持ち込んで微減。
- **C（残差スロット選択）は棄却**: 強度を緩めても全滅。**passage 丸ごとの直交射影は粗すぎ**、
  関連まで消す（MMR と同じ過剰多様化）。「Stage 1 の残差演算は entity 粒度でのみ効き、
  passage 粒度では成立しない」＝統一原理は passage レベルでは不成立、という否定的知見。

### 文脈効率（ゴミの少なさ）— LLM フリー、`research/recall_curve.py`（400q, γ=0.5）

e2e 回答精度は reader LLM の能力を交絡させるため retrieval の分離評価には不適。代わりに
「goldを上位に詰め、少ないチャンクで済むか」を LLM なしで直接測る（score = PPR 順 vs coverSoft）。

| 指標 | 2wiki | musique | hotpot |
|---|---|---|---|
| Recall@1  score→coverSoft | 33.1→**38.2** | 19.7→**23.7** | 20.4→**30.4** |
| Recall@3  score→coverSoft | 57.2→**62.9** | 30.2→**38.9** | 42.2→**58.2** |
| Precision@1 score→coverSoft | 74.5→**86.5** | 47.8→**57.2** | 40.8→**60.8** |
| min-k(all gold) 中央値 score→coverSoft | 4→**3** | 17→**12** | 7→**5** |

- **Precision@1 が全データセットで大幅改善**（hotpot 40.8→60.8）＝先頭チャンクが gold である率が上がる
  ＝**トップのゴミが減る**。Recall@k は低 k ほど差が大（hotpot Recall@3 +16pt）＝上位に詰まる。
- **min-k 中央値が減少**（両 gold を揃えるのに読むチャンク数が減る）。「深さ50内で解ける率」は
  両者同一（同じ候補プールを順序だけ変えている）。
- 正直な但し書き: 2wiki の min-k 平均は 9.5→11.4 とやや増（中央値は改善）＝典型は締まるが
  一部で 2 つ目 gold を深部へ押しやる裾。中央値（頑健）は全改善、平均は裾依存。
- 含意: retrieval の主張は **e2e 不要**、LLM フリーの効率指標で「閉じて」示せる。e2e はやるとしても
  「recall 改善が reader を毒していないか」の安全弁として最小限に留める。

## 一貫した原理としての位置づけ（論文の背骨）

Stage 1 残差伝播（[research-direction](2026-07-03-research-direction.md)）と合わせると、
「**検索の状態で次の判断を条件付ける**」という単一原理が両ステージを貫く:

- Stage 1: これまでカバーしたクエリ成分（残差）で次ホップの σ を条件付ける
- Stage 2: これまで選んだパッセージ集合で次の 1 枠の利得を条件付ける

両方ともトークンフリー・訓練フリー・行列演算のみ。

## 既存研究との差分

| 手法 | 多様化の基準 | LLM/訓練 | うちとの差分 |
|---|---|---|---|
| **MMR** (Carbonell&Goldstein 1998) | 埋め込みの汎用類似度 | 不要 | 埋め込み多様性でなく**グラフ活性化質量のカバレッジ**。L_ei で Stage 1 と接続。**対照実験で MMR を同条件で回し、活性化カバレッジが上回るか**を示す＝「グラフが効く」証明 |
| **BDTR** (2509.25530) | ブリッジ証拠を上位へ | LLM思考生成 | 問題意識（順位押し上げ）は同じ、処方が構造のみ |
| **BridgeRAG** (2604.03384) | ブリッジ条件付きペア選択 | LLM judge必須 | オンライン LLM ゼロ |
| **PropRAG** (EMNLP25) | 命題パスの beam search | LLM命題抽出 | 表現側も LLM フリー |
| Set-based IR / facility location | 被覆最大化 | — | 被覆の重みを**クエリ活性化**に置く点が新規 |

## 成功基準（先験的に宣言）

- MuSiQue AllGoldHit@5: 7.0% → **14% 以上**（再ランキング上限 20.3% の半分超）
- 2wiki AllGoldHit@5: 28.7% → **34% 以上**
- 同条件の MMR 対照を上回る
- レイテンシ増ほぼゼロ（候補は既に計算済みの PPR スコア上位）
