# 研究方向性: 残差クエリ伝播による関係フリーGraphRAGの多ホップ強化

2026-07-03 ・ 失敗分析（[2026-07-03-failure-analysis.md](2026-07-03-failure-analysis.md)）に基づく研究計画

## 1. リサーチクエスチョン

> 関係フリーグラフ上の活性化伝播に「ホップ状態」を持たせることで、
> LLM呼び出しゼロのまま、LLM支援型検索（BridgeRAG等）との多ホップ性能差をどこまで詰められるか？

### 根拠（実測）

- LinearRAG失敗の60%（MuSiQue）/ 42%（JEMHopQA compositional）は **under-propagation**:
  伝播がシードで死に、ブリッジを渡れない
- 33〜50%は逆の **over-propagation**: 数百エンティティが活性化して正解が埋没
- 原因は共通: 式5のσ（文の通りやすさ）が**クエリ全体との類似度で最初に1回だけ固定**され、
  「今どのホップで、何をカバー済みか」を持たない
- δ（閾値）の調整では解けないことを実証済み: ACTIVATION失敗の復活率はδに対して**非単調**
  （最良δ=0.3でも36%、大域変更はchurn 14%でnet +2%のみ）

## 2. 提案手法（working name: **ResiRAG** — Residual-query Propagation on Relation-free Graphs）

### C1: ホップごとの残差クエリσ再計算（主貢献）

各ホップtで、既にカバーしたクエリ成分を埋め込み空間で除去し、残差でσを測り直す:

```
q_0 = q（クエリ埋め込み）
各ホップ t:
  σ_t(s) = cos(emb_s, q_t)                    # 文の通りやすさをホップごとに更新
  伝播:  a_t = max(M^T (σ_t ⊙ (M a_{t-1})), a_{t-1})
  カバー更新: U_t = span{ 新規活性エンティティ／マッチ文の埋め込み }
  q_{t+1} = normalize( q_t − proj_{U_t}(q_t) )   # 直交射影で「済んだ部分」を差し引く
```

- **under-propagation対策**: 「大橋和也の出身都道府県の県庁所在地」で hop1後に
  「大橋和也」成分が消え、残差が「都道府県・県庁所在地」に集中 → ブリッジ文のσが相対的に浮上
- **over-propagation対策**: カバー済み方向と平行な文（同じ話の反復）はσが落ち、自然に枝刈り
- 追加コストは1ホップあたり行列ベクトル積1回 + 射影1回。**線形性・トークンフリー性を完全維持**
- 古典との接続: Rocchio(1971)の負フィードバックを「文書」でなく
  「グラフ活性化のホップ状態」に適用する再解釈。IRの スプレッディング・アクティベーション
  (Crestani 1997) に情報論的な抑制を入れる、と位置づける

### C2: 絶対閾値δの廃止 → ホップ相対予算（副貢献）

δ非単調性の実測が示す通り、絶対閾値は原理的に両モードを両立できない。
「各ホップで残差σスコア上位m件のみ活性化」（ホップ予算）または
「そのホップの最大活性に対する相対閾値」に置換。C1と直交する改善なのでablationで分離。

### C3(optional): 活性化統計による無訓練アダプティブ融合

活性エンティティ数・シード類似度ギャップ等のグラフ内部信号から式7のλを質問ごとに調整。
単一ホップ回帰（JSQuAD −6.9pt）対策。分類器訓練不要な点がAdaptive-RAGとの差分。

### C4(optional/日本語): エンティティ埋め込み+NERタイプによるエイリアス正規化

表記ゆれ統合。JEMHopQAでの効果をablationとして報告。

## 3. 既存研究との差分（ここが審査の生命線）

| 手法 | グラフ構築 | 検索時LLM | ホップ状態 | うちとの差分 |
|---|---|---|---|---|
| **LinearRAG** (ICLR26) | NERのみ・トークンフリー | 不要 | **なし**（σ固定・絶対δ） | 我々のベース。σ固定が失敗の60%の原因だと実証し、ホップ状態を導入する |
| **SPRIG** (2602.23372) | NER共起・トークンフリー | 不要 | なし（PPRのみ） | 「安いグラフ構築」の主張は同じ土俵 → **構築でなく伝播アルゴリズムで差別化**。要26テーブル精読 |
| **HippoRAG 2** | LLMでOpenIEトリプル | 不要 | なし | 構築にLLM必須（トークン線形費用）。関係抽出エラーに感度 |
| **PropRAG** (EMNLP25) | **LLMで命題抽出** | 不要（beam search） | パス探索で保持 | パス発見の思想は近いが、命題抽出にLLM必須。我々は表現側もLLMフリーのまま「ホップ状態」だけ輸入 |
| **BridgeRAG** (2604.03384) | 不要（graph-free） | **LLM judge必須** | ブリッジ条件付き | R@5 SoTA (MuSiQue 0.81)。オンラインでトークン費用。我々の目標は「LLMゼロでこれに何pt差まで迫るか」 |
| **BDTR** (2509.25530) | 既存GraphRAGに載せる | LLM thoughts | — | ブリッジ証拠の順位押し上げが要という**診断**を共有。処方がLLM思考生成 vs 我々は埋め込み演算のみ |
| **MDR** (Xiong+21) | グラフなし（密検索） | 不要だが**要訓練** | 再エンコードで保持 | ホップ条件付けの最近接先行。パッセージ連結を**再エンコード**（訓練済みbi-encoder必須・2hop固定）。我々は訓練ゼロ、既製埋め込みの**ベクトル演算だけ**でホップ状態を作り、パッセージでなくエンティティ粒度 |
| **IRCoT / GoldEn** | — | LLMでクエリ再生成 | 自然言語で保持 | クエリ再構成をLLM生成でやる系譜。我々は埋め込み空間の射影で代替 |
| **Adaptive-RAG** | — | — | — | ルーティングに訓練済み分類器。C3は無訓練・グラフ内部信号 |
| **Rocchio / PRF** (古典) | — | — | — | 負フィードバックの祖。文書単位・単発。ホップ毎のグラフ活性化への適用は新規 |

**新規性の一文**: 「訓練もLLM呼び出しも一切なしに、関係フリーグラフの活性化伝播へ
ホップ状態（クエリ残差）を導入した最初の手法」。
表現学習（MDR/GRITHopper）・LLM抽出（PropRAG/HippoRAG）・LLM判断（BridgeRAG/IRCoT）の
いずれのコストも払わない象限が空いている。

## 4. 評価計画

- **データセット**: MuSiQue / 2Wiki / HotpotQA（英・比較可能性）+ **JEMHopQA**（日本語multi-hop、
  英語圏の先行が評価していない差別化ポイント）+ JSQuAD（単一ホップ回帰の監視）
- **指標**: gold-passage Recall@5 を主指標に**追加**（PropRAG/BridgeRAGと同じ土俵）。
  Contain@5は継続（NOT_IN_CORPUS 27%は指標ノイズと判明済み）
- **ベースライン**: LinearRAG(自前再現), naive dense, 論文数値としてHippoRAG2/PropRAG/BridgeRAG
- **ablation**: C1のみ / C2のみ / C1+C2 / +C3 / +C4、ホップ数別・質問タイプ別の内訳
- **効率**: クエリあたりレイテンシとメモリ（線形性の維持を示す）
- **成功基準（先験的に宣言）**:
  - MuSiQue ACTIVATION失敗バケットの復活率 > 36%（δ最適化の上限を機構で超える）
  - MuSiQue全体 hit@5 で +5pt以上、churnを示して「復活 >> 新規喪失」
  - 単一ホップ（JSQuAD）を劣化させない（C3込み）

## 5. リスクと先に潰すこと

1. **残差の意味汚染**: エンティティ埋め込みの引き算は意味を綺麗に除去しない可能性
   → 射影対象を「マッチした文の埋め込み」にする変種、射影強度をマッチ類似度でスケールする変種を最初から用意
2. **SPRIG重複**: 26テーブルを精読して主張の重なりを正確にマップ（最優先の宿題）
3. **BridgeRAGとの絶対差**: R@5 0.81は高い。「LLMゼロ縛りでの最良」+「コスト曲線」で立ち位置を作る
4. 埋め込みモデル依存性: mpnet以外（BGE-M3等）でも再現するかを1本チェック
